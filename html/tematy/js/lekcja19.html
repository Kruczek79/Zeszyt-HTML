Temat: Obiektowy JS - OOP

  1. Co to sa paradygmaty programowania


    Paradygmaty programowania – to różne sposoby, w których dany program
    lub język programowania może być zorganizowany. Każdy paradygmat składa się
    z pewnych struktur, cech i opinii na temat tego, jak należy
    rozwiązywać typowe problemy programistyczne.

  2. Co to OOP


    Programowanie obiektowe (ang. OOP — Object Oriented Programming) to popularny paradygmat programowania,
    w którym modelujemy istniejącą rzeczywistość za pomocą obiektów, zamiast stosować tylko funkcje i logikę.
    Projektowanie obiektowe polega na umiejętności przedstawienia sytuacji z życia codziennego pod postacią
    obiektów i relacji między nimi.

  3. Klasa i Obiekty


    1. Obiekty


      Obiekt – wyodrębniony element rzeczywistości, element może być materialny
      lub abstrakcyjny Obiekt posiada wlasciwosci(pola) i funkcje(metody) 
      //tworzenie obiektu
const programista = {
    kolorOczu: 'zielony',
    jezykiProgramowania: ["js","c++","c#"],
    programuj: function(){console.log("programuje");}
}

    2. Klasy


      Klasa to przepis(pattern) na stworzenie obiektu. Kazdy obiekt klasy posiada wlasciwosci
      i metody danej klasy. Klasa rowniez posiada swoje wlasciwosci(pola) i funkcje(metody)
      oraz posiada swoj konstrukor - czyli metoda wywolywana w trakcie stworzenia obiektu
      jak i rowniez posiada pola statyczne - czyli takie ktore sa przypisane do klasy(nie obiektu) 
      //klasa programista
class programista{
//pola klasy
    static liczbaProgramistow;
    kolorOczu;
    jezykiProgramowania;
//metody klasy
    programuj(){
        console.log("programuje");
    }
//konstruktor
    constructor(){
        this.kolorOczu = "zielony"; //ustawianie wartosci na "zielony"
        this.jezykiProgramowania = ["js","c++","c#"];
    }
}
//tworzenie obiektu z klasy
const obiekt = new programista();

  4. Wazne paradygmaty programowania obiektowego

    1. Enkapsulacja


      Enkapsulacja to proces hermetyzacji pol i metod klasy aby
      nikt nie mogl ich zmienic przypadkiem. Uzywa sie do tego klauzul okreslajacych dostep:

      1. Private


        Do prywatnych pol ma dostep jedynie klasa i tylko i wylacznie ona.
        Nazwy pol i metod poprzedzamy znakiem "#"         
                                                        class programista{
    #prywatnePole;
    #prywatnaMetoda(){
        //kod
    }
}

      2. Public


        Do publicznych pol ma dostep kazdy 
                                                    class programista{
normalnePole;
normalnaMetoda(){
    //kod
}
}

      3. Protected


        Do protected ma dostep jedynie klasa oraz klasy dziedziczace z niej.
        Nazwy pol i metod poprzedzamy znakiem "_"         
                                                        class programista{
    _protectedPole;
    _protectedMetoda(){
        //kod
    }
}

    2. Dziedziczenie


      Dziedziczenie jest mechanizmem, w którym można stworzyć nową klasę,
      bazując na już istniejącej (nadrzędnej) klasie tzw. superklasie.
      Klasa matka dzieli zestaw publicznych lub protected pol i metod z klasą dziecka.
      Dziedziczenie w JS okreslamy przed dodanie po nazwie klasy "extends" i nazwe klasy z ktorej dziedziczy       
                                              class programista extends Informatyk{
}

    3. Abstrakcja


      Klasa abstrakcyjna różni się od zwykłej klasy tym,
      że nie można utworzyć jej instancji. Może jednak dostarczyć metod zdefiniowanych i abstrakcyjnych,
      które muszą zostać zdefiniowane w klasach potomnych. Jednak tego nie uzywamy w JS

    4. Polimorfizm


      Polimorfizm to zdolność obiektu do przybierania różnych form.
      Występuje np., w sytuacji, gdy korzysta się z wielu klas,
      powiązanych ze sobą poprzez dziedziczenie. Sprowadza się to do tego, że polimorfizm pozwala
      na wykonanie wielu implementacji poprzez zdefiniowanie jednej klasy.
      Przyklad polimorfizmu:
