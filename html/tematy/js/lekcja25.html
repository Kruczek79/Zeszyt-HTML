Temat: == vs === Koercja typow i operatory porownania

  1. Operatory porownania


    Operatory porownania to operatory sluzace do porownania wartosci.
    Zwracaja true albo false. Wyrozniamy dwa rozne operatory:

    1. ==


      Operator sluzacy do porownania wartosci.
      Podczas uzywania jego dochodzi do tzw koercji typow
      w przypadku gdy wartosci sa roznego typu(np:. number i string)
      czyli do zamiany typow aby byly jednakowe       
                                              const a = "21";
const b = 0;                           
a == b; //porownuje wartosci i zachodzi koercja typow

    2. ===


      Operator sluzacy do porownania strikte wartosci.
      Porownuje typy wartosci jak i same wartosci.
      Nie zachodzi w nim koercja typow       
                                          const a = 1;
const b = 0;                           
a === b; //porownuje wartosci i typy

  2. Koercja typow - przebieg


    Koercja wykonuje sie dopoki nie zwroci true lub false(w petli)
    Za kazdym razem gdy cos zwraca nastepuje powrot do poczatku petli
    1. a(number) == b(number) => zwraca a===b
    2. null == undefined => zwraca true
    3. undefined == null => zwraca true
    4. a(number) == 'b'(string) => zwraca a==Number('a')
    5. 'a'(string) == b(number) => zwraca Number('a') == b
    6. a(boolean) == b(any) => zwraca Number(a) == b
    7. a(any) == b(boolean) => zwraca a == Number(b)
    8. a(number, string) == b(Object) => zwraca a == String(b)/Number(b)
    9. a(Object) == b(number, string) => zwraca String(a)/Number(a) == b
    10. zwraca false
