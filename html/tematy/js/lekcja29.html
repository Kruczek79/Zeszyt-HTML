Temat: Klonowanie obiektow - Wartosc a referencja

  1. Wartosc a referencja


    Jezeli jakies zmiennej przypisujemy wartosc prymitywna(nie obiekt) to przypisywana jest do niej wartosc nie referencja.
    A jesli przypisujemy zmiennej wartosc nieprymitywna(obiekty, tablice) to przypisywana jest do niej referencja     Referencja to odwolanie do tego samego miejsca w pamieci co zmienna do ktorej referujemy. 
    let a = 0;
let b = a; //b = 0

a = 1;
b = 0; //b bedzie dalej 0 poniewaz a jest wartoscia prymitywna


    let a = {name: 'John'};
let b = a; //b = {name: 'John'}
a.name = "Bob"; //b = {name: "Bob"} poniewaz obiekt jest wartoscia nieprymitywna wiec b jest referencja do a

  2. Klonowanie obiektow


    Aby uniknac sytulacji gdzie wystepuja niechciane referencje kozystamy z klonowania obiektow
    Istnieja 3 rozne sposoby na to

    1. Kopia z uzyciem spread operator

      Sluzy do kopiowania plytkiego(wiekszosc danych to wartosci prymitywne) 
      const John = {
name: "John"
}
const Bob = {...John} //kopiuje wartosci prymitywne przez co niema referencji

    2. Kopia z uzyciem JSON

      Uzywamy gdy w obiekcie nie posiadamy metod(funkcji) oraz typow obiektow(np obiektu typu date) 
      const John = {
name: "John",
skill: {name: "Fireball", level:5}
}
const Bob = JSON.parse(JSON.stringify(John)) 
//zmienia obiekt john na jsonowy string a nastepnie zmienia spowrotem na obiekt js
//(json nie posiada referencji wiec one w trakcie sie kasuja)

    3. Kopia z uzyciem spread operator 2

      const John = {
name: "John",
skill: {name: "Fireball", level:5}
}
const Bob = {...John, skill:{name: John.skill.name, level: John.skill.level}}
//kopiuje wartosci prymitywne przez co niema referencji
